# THE UNDERVAULT — Design ⇄ Code Handoff Contract
**Version 1.0 · Read by BOTH Claude Design and Claude Code · Companion to 04-UI-SPEC**

---

## 1. The Model: Nobody "Integrates" — Both Conform

`04-UI-SPEC.md` is the single contract. **Claude Design renders it** (prototypes, mocks, icons, tokens). **Claude Code implements it** (working screens). The human operator moves files between them and arbitrates. Neither tool ever consumes the other's *reasoning* — only the files defined below. If either tool wants to deviate from 04, it proposes the change in writing (see §7); it never just deviates.

## 2. Directory Contract (the only shared surface)

```
repo/
├─ design/                          ← Claude Design's deliverables land here
│  ├─ tokens/
│  │  ├─ tokens.css                 ← SHIPPED: imported by DOM surfaces
│  │  └─ tokens.ts                  ← SHIPPED: imported by Phaser code
│  ├─ prototypes/                   ← runnable HTML/CSS (some near-production, see §4)
│  │  ├─ guildhall.html             (all 3 states via ?state= query or class toggle)
│  │  ├─ sheet-waystone.html  sheet-epitaph.html  codex.html  vision-gate.html
│  ├─ mocks/                        ← reference images (never shipped)
│  │  └─ scr-{screen}-{state}.png   e.g. scr-descent-lowwax.png
│  ├─ components/
│  │  └─ cmp-{name}-{state}.png     e.g. cmp-sealbutton-pressed.png
│  ├─ icons/                        ← SVG, 24 px grid, stroke=currentColor
│  ├─ death-card/                   ← layered template + zones.json (text injection boxes)
│  └─ CHANGELOG-design.md           ← one line per change (what + why + 04 section)
└─ src/client/assets/               ← production art (atlases, bitmap fonts, audio) — Code-owned
```

Naming is binding: `scr-` and `cmp-` prefixes with the exact state names from 04 §3–4, kebab-case. Claude Code treats `design/` as **read-only input**; Claude Design never touches `src/`.

## 3. The Three Crossing Surfaces

1. **Tokens** — `tokens.css` + `tokens.ts` generated from 04 §2.1–2.3. Design edits values *only here*; Code imports (hardcoding hex/px is an invariant violation per CLAUDE.md). One source, two exports, zero drift.
2. **Assets** — icons (SVG → inlined in DOM; baked into a texture atlas for Phaser by `tools/atlas`), death-card template + `zones.json`, prototypes.
3. **Visual targets** — mock PNGs. Claude Code opens them, implements to match, and attaches its own screenshot per PR (existing DoD). The human compares; pixel-perfection is required for DOM surfaces, "faithful within engine constraints" for Phaser surfaces.

## 4. Surface-by-Surface Integration Map

| Surface | Runtime tech | Design delivers | Code consumes it by… |
|---|---|---|---|
| **Guildhall** (inline entry) | Pure DOM | `prototypes/guildhall.html` — near-production | Adapting ~1:1 into `splash.html`: strip prototype toggles, wire postData, keep markup/CSS |
| **Waystone / Epitaph / Codex / Vision gate / menus** | **DOM overlay over the Phaser canvas** (decided — crisp text, real `<input>` for last words, accessibility) | Prototypes (HTML) | Mounting as overlay layer components; forms wired to `/api/*` |
| **Descent HUD + world** | Pure Phaser (per-frame) | Mock PNGs + measurement callouts | Re-implemented in Phaser reading `tokens.ts`; mock = visual target only (HTML cannot cross into canvas) |
| **Death card** | Runtime PNG compositor | Layered template + `zones.json` | Compositor draws template, injects text into zones, uploads via media API |
| **Icons** | Both | SVG set | DOM: inline SVG; Phaser: atlas-baked via `tools/atlas` |
| **Match-strike, world FX** | Pure Phaser | Storyboard frames (optional) | Reference only |

## 5. Game Art Is Out of Design Scope (placeholder-first strategy)

Claude Design produces UI, not 16 px sprite animation or normal maps. Plan of record: **Claude Code ships token-conformant programmatic placeholders** (shaped, lit, readable) for all world art at M4 so nothing blocks; real pixel art is sourced by the operator (artist / asset iteration) and dropped into `src/client/assets/` matching the manifest in 03 §3 — the atlas pipeline ingests either without code changes. Bitmap fonts (`sans-8`, `serif-12`, `glyph-12`) are generated by `tools/` from font sources, not hand-drawn by Design.

## 6. The Operating Loop (human is the bus)

1. Design sprint against 04 §8 checklist → deliverables into `design/` per §2 naming.
2. Operator reviews vs 04 (5-minute pass: tokens match? states all present? copy verbatim from §7?).
3. Commit. 4. Code implements the affected screens; PR includes side-by-side screenshot.
5. Operator eyeballs the diff; optionally feeds Code's screenshot **back to Claude Design** with "critique vs your mock" — the cheapest QA in the pipeline.
6. Divergences resolve by editing the *contract* (04 / tokens), never by verbal agreement.

**Timing:** fully parallel. Code M1–M3 (engine) needs nothing from Design. First hard dependency is M4 (Guildhall + HUD). Design sprint 1 therefore = tokens + icons + `guildhall.html`; sprint 2 = sheets + Codex; sprint 3 = death card + component sheets + mock passes.

## 7. Change Control

Any change to a token value, layout, or copy string = edit the contract artifact (`tokens.*`, or propose a 04 diff) **plus** one line in `CHANGELOG-design.md`. Claude Code checks that changelog at session start; unexplained visual drift in mocks without a changelog line is ignored in favor of 04. Ties are broken by the operator, recorded in 04.
